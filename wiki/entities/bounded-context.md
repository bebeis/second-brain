# 바운디드 컨텍스트 (Bounded Context)

모델이 완전한 의미를 갖는 경계를 가진 문맥. 같은 용어라도 하위 도메인마다 의미가 다르므로, 모델을 하나로 뭉치지 말고 컨텍스트로 경계 지어 분리한다.

## 성질
- 모델의 경계를 결정하고, **논리적으로 한 개의 모델**을 가진다.
- 주로 **용어를 기준으로** 구분된다.
- **기능을 제공하는 물리적 시스템**이며 도메인 모델은 그 안에서 구현된다.
- 이상적으로 하위 도메인과 1:1이지만 현실은 **조직 구조**에 좌우된다. 물리적으로 하나여도 패키지로 논리적 경계를 유지한다.

## 컨텍스트 간 관계
- **고객/공급자(하류/상류)**, **오픈 호스트 서비스**(단일 API 공개), **안티코럽션 계층**(외부 모델 침범 차단), **공유 커널**(모델 공유), **독립 방식**(수동 통합).
- **컨텍스트 맵**: 경계·관계·하위 도메인·조직을 한눈에 그린 지도.
- 마이크로서비스와 잘 어울린다(컨텍스트별 모델 분리·독립 배포).

## 통합
직접(REST API)·간접(메시지 큐, 출판/구독). 강결합은 [도메인 이벤트](../topics/ddd-domain-events.md)로 낮춘다.
